Contributing to OneHeart
=======================

Thanks for your interest in contributing! We welcome contributions of all kinds: bug reports, documentation improvements, design tweaks, and new features.

How to Contribute
-----------------

1. Fork the repository and create a branch using the `feature/` or `fix/` prefix (e.g., `feature/add-gallery`).
2. Make changes on your branch and include tests where applicable.
3. Follow Conventional Commits for commit messages (e.g., `feat(achievements): add back button`).
4. Open a Pull Request against the `main` branch with a clear description of the change.

Local Development
-----------------

- Install dependencies: `npm install` or `pnpm install`.
- Create a copy of `.env.local.example` as `.env.local` and fill in required environment variables.
- Run the development server: `npm run dev` or `pnpm dev`.

Reporting Security Issues
-------------------------

If you find a security vulnerability, please report it privately to `davemeshak_j@srmap.edu.in` and do not open a public issue.

Code Style
----------

- Follow existing code patterns and Tailwind utility classes.
- Run linters and formatters before opening a PR: `npm run lint` and `npm run format` (if available).

Thank you for helping make OneHeart better!
